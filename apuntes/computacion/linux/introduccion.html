Introducción a Linux.

26 de abril del 2021

Cuando estamos metidos en el mundo de la programación, no es raro escuchar sobre linux, pero ¿debería considerar instalar linux? Para eso es este artículo para exponer razones para considerar linux, un resumen de conceptos claves para entender la terminología, además de un apéndice para tener una base de cómo funciona linux a un nivel más abstracto.

¿Porqué considerar Linux?

Yo uso linux por que me gusta entender cómo funciona mi computadora, y me gusta tener la capacidad de personalizarla lo máximo posible, porque si alguien más decide sobre mi computadora (cómo decidir cuando se actualiza) entonces no sentiría que me pertenece del todo y que feo no ser dueño de algo que compraste.

Optimización

Ahora, entiendo que no todos piensas igual a mí, por ejemplo, puedes usar linux sí tienes una computadora de pocos recursos ya que optimiza mejor los recursos y tiene menos procesos (o tareas) ejecutándose de fondo que un sistema Windows; de Mac no puedo hablar porque no lo he usado.

Automatización

Por otro lado en linux, y en cualquier computadora, todo lo que haces son comandos que se ejecutan: cuando aprietas un botón, ejecutas un comando; Cuando abres una aplicación, estás ejecutando un comando. Por lo que linux te da la capacidad de entender esos comandos y automatizarlos en un script, cómo el que uso para generar esta página.

Seguridad

Por último hay una capa extra de seguridad a la hora de trabajar con linux ya que tiene un sistema donde cada archivo le pertenece a un usuario y solo se ejecuta un programa si le das permiso de que se ejecute.

Partes de un sistema Linux

Ahora, Un sistema operativo linux básicamente se conforma del kernel, la terminal y los binarios. Los binarios son las aplicaciones que se ejecutan, normalmente se encuentran en la carpeta /bin y suelen estar escritos en C, por lo que están bastante optimizados.

Shell

El shell, es el programa con el que te comunicas con la computadora, casi siempre tu shell por defecto es bash sin embargo hay varias alternativas interesantes. Aquí puedes ejecutar comandos y comunicar comandos entre sí, y ejecutar tus propios programas para automatizar estos comandos.

Terminal

La terminal es un concepto, no un programa, este concepto básicamente define al intermediario entre el usuario con la computadora, por lo que incluye, el teclado la pantalla, el kernel, las aplicaciones y el shell.

Emulador de Terminal

Esta es la aplicación gráfica que ejecuta la shell, aquí es donde escribiremos nuestros comandos, daremos permisos a scripts, ejecutaremos programas, además claro de compilar desde la terminal, básicamente aquí sacaremos toda la potencial de linux.

Sistema operativo

El sistema operativo, en linux es el conjunto de aplicaciones, cada sistema operativo escoge aplicaciones a partir de su filosofía, es por eso que Ubuntu es tan recomendada ya que tiene una selección de aplicaciones básicas para el usuario promedio, y los sistemas basados en archlinux tienen el mínimo de aplicaciones, osea, tu instalas y configuras el entorno gráfico, porque su público es distinto.

Kernel Linux

Por último el kernel linux es lo que comunica los componentes electrónicos con los ejecutables. Es por eso que se llama sistema operativo GNU/linux, linux es el kernel y las utilidades de GNU son aplicaciones muy simples pero bastante útiles para hacer algo más grande.

Apendice: Sistema de Archivos de linux

Es fácil escuchar que En linux todo es un archivo, ya que su forma de abstraer las cosas es así, las aplicaciones son binarios que entiende la computadora y son archivos que se guardan en la carpeta /bin.

Cuando ejecutamos un programa y se queda ejecutándose de fondo es un proceso, que tiene una id que podemos ver ejecutando el comando ps ax. Estos procesos se encuentran en el directorio /proc.

Algunos programas requieren configuraciones extras que se pueden encontrar en /etc sin embargo los usuarios tienen sus directorios personales en /home y sus configuraciones personales en /home/.config. El directorio personal del usuario root es /root

Los dispositivos físicos, sean nuestros discos duros, cámaras y usb que conectemos se encuentran en la carpeta /dev por ejemplo, podemos ver nuestra cámara con ffplay /dev/video0 o nuestros discos duros y sus particiones con el comando lsblk.
